# Project Documentation: repo-onboarding-agent documentation

## 1. Project Overview
- **Description:** This project is an automated code documentation pipeline. It analyzes a given Python Git repository by cloning it, parsing the source code to build an Abstract Syntax Tree (AST), and constructing a call graph to understand the relationships between different code components. This structured data is then processed by a series of Large Language Models (LLMs) to generate detailed, human-readable documentation in Markdown format.
- **Key Features:**
  - Clones remote Git repositories for local analysis.
  - Parses Python code into an Abstract Syntax Tree (AST) to identify functions, classes, and their structures.
  - Builds a call graph to map dependencies and interactions between code components.
  - Utilizes a dual-LLM architecture: a "Helper LLM" for detailed code analysis and a "Main LLM" for synthesizing the final report.
  - Automatically generates comprehensive technical documentation in a structured Markdown format.
- **Tech Stack:**
  - langchain
  - langchain-core
  - langchain-google-genai
  - google-generativeai
  - pydantic
  - networkx
  - GitPython
  - python-dotenv

## 3. Use Cases & Commands
The primary use case for this agent is to generate technical documentation for a Python-based Git repository. The main entry point is the `main_workflow` function within `backend/main.py`.

**Primary Command:**
The agent is executed by running the main script. It takes a Git repository URL as input and produces a Markdown documentation file as output.

1.  **Configure Environment:** Create a `.env` file based on `.env.example` and add your Google Gemini API key.
2.  **Run the analysis:**
    ```bash
    python backend/main.py
    ```
    *(Note: The repository URL is currently hardcoded in `main.py` for testing purposes but is designed to be provided dynamically.)*

The script will then perform the full analysis pipeline and save the resulting report in the `result/` directory with a timestamped filename (e.g., `report_DD_MM_YYYY_HH-MM-SS.md`).
